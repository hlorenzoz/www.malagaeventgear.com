# Changelog - Malaga Event Gear (MEG) Website

## [Unreleased]

### Changed
- **Lighthouse CI Command Report Auto-Cleanup**: Modified the `test-lighthouse` recipe in `justfile` to run `rm -rf .lighthouseci` prior to running the Lighthouse auditor. Also updated the interactive command definitions in `.agents/commands/test-lighthouse.md` and `.claude/commands/test-lighthouse.md` to explicitly state and execute this cleanup step, ensuring only the freshest audit reports are stored.

### Added
- **Full PWA & SEO Favicon Suite Generation**: Generated optimized raster icons (`favicon.ico` at 32x32, `favicon-16x16.png`, `favicon-32x32.png`, `apple-touch-icon.png` at 180x180, `icon-192.png` at 192x192, and `icon-512.png` at 512x512) directly from the user's `static/favicon.webp` utilizing macOS native `sips` tool.
- **Global Favicons & PWA Setup in app.html**: Integrated all static favicons, Apple touch icon, theme-color metadata, and PWA manifest links globally inside `src/app.html` for instant cross-device caching and PWA installs.
- **Vector Favicon Copy**: Copied `$lib/assets/favicon.svg` to `static/favicon.svg` to serve it as a static fallback at the root level.

### Fixed
- **Cleaned Up Duplicate Head Icon Links**: Cleaned up the redundant favicon import and `<svelte:head>` icon block from `src/routes/+layout.svelte` and removed the duplicate manifest link from `src/lib/components/seo/SeoHead.svelte` to optimize crawler efficiency and eliminate duplicate network declarations.
- **Heading Sequence Conformity on Contact Page**: Changed the sidebar "Contact Details" title from `<h3>` to `<h2>` in `contact/+page.svelte`. This resolves the WCAG/SEO heading level skipping issue where the page jumped from `<h1>` to `<h3>` without any intervening `<h2>` headings, establishing a clean sequentially-descending hierarchy.
- **Language Toggle Accessibility Label Compliance**: Refactored the `aria-label` attribute on the language toggle button in `TopNavBar.svelte` to prepend the visible text label ("ES - Change language to Spanish" / "EN - Change language to English"). This resolves the WCAG `label-content-name-mismatch` rule, ensuring screen readers receive an accessible name that begins with the visible button text.
- **Material Symbols Asynchronous Restructure**: Resolved icon rendering failures on internal pages (e.g. `/equipment/`, `/packages/`) caused by the full removal of Material Symbols. Integrated the Material Symbols stylesheet back into `+layout.svelte` using an asynchronous print stylesheet pattern with client-side JavaScript activation, preserving Green Core Web Vitals performance while restoring universal icon support across the entire catalog and testimonials.
- **Google Fonts Compiler Crash Fix**: Refactored the inline `onload="this.media='all'"` event in `+layout.svelte` into a safe `<script>`-based client-side execution block. Inline HTML event handlers inside Svelte components cause compilation failures under Svelte 5 and Rolldown/Vite 6 parsers in Cloudflare's build environment.
- **Cloudflare Build Crash Mitigation**: Adjusted the `package.json` build script to run `wrangler types` instead of `wrangler types --check`. This prevents deployment build failures caused by environment, hash, or platform-specific micro-version mismatches of generated type definitions in Cloudflare's CI/CD pipelines, while keeping type safety active.
- **Commitment of Static Images**: Removed `/static/images/` from `.gitignore` to ensure the package and service images (such as `/images/packages/eco.webp` and `/images/services/projectors-screens.webp`) are tracked and uploaded. This resolves a prerendering 404 error during Cloudflare deployments.
- **Relocated _redirects to Root**: Moved `_redirects` from `static/` to the project root directory. `@sveltejs/adapter-cloudflare` requires `_redirects` to sit at the root level to adapt edge redirects correctly, otherwise the deployment fails during SvelteKit build.

### Added
- **Lighthouse Reports Ignored**: Configured `.gitignore` to dynamically exclude raw Lighthouse JSON and HTML local diagnostic reports (such as `/lighthouse-report.json`) and `.lighthouse/` temp directories from git tracking, keeping the repository clean.
- **Centralized SVG Icon System**: Implemented a highly optimized, reactive Svelte 5 `<Icon />` component (`src/lib/components/navigation/Icon.svelte`) featuring inlined SVG pathways. This completely eliminates the synchronous, render-blocking Material Symbols Outlined font from the network path, yielding **3.95 MB of payload savings** (a **100% reduction** in icon asset size).
- **Asynchronous Non-Blocking Google Fonts Loading**: Swapped synchronous stylesheet links in public layout for an asynchronous print stylesheet pattern with preconnect hints. This bypasses the **20.1-second external render-blocking network delay** on mobile viewports, allowing instant fallback font swapping.
- **Responsive WebP Hero Assets**: Converted the raw `/premium_event_stage.png` background (**773 KB**) to optimized WebP format (**83.9 KB** full, **52.1 KB** mobile crop). Integrated a native Svelte `<picture>` tag to conditionally serve the mobile crop, reducing the mobile Hero payload by **93%** and improving LCP.
- **Localized Bento Services Assets**: Downscaled and saved bento grid images locally in `static/images/services/` as compressed WebPs (~15-45KB each), avoiding external negotiation with `images.unsplash.com` and saving over **1.7 MB** of payload.
- **Desactivación de Indexación para Bots de Google**: Configurado el archivo `static/robots.txt` para desautorizar explícitamente a `Googlebot` y al resto de motores de búsqueda (`*`) el rastreo e indexación del sitio web, protegiendo el entorno de staging/desarrollo.
- **Optimización de Exclusiones en .gitignore**: Añadidos patrones para ignorar dinámicamente los archivos generados por wrangler (`*-export.ts`) y el directorio temporal local de imágenes (`/static/images/`), manteniendo limpio el flujo de control de git.
- **Renombrado del Catálogo a /equipment/**: Cambiado el directorio físico de SvelteKit de `/services/` a `/equipment/` para alinear la semántica de navegación del catálogo de equipamiento del sitio de MEG, actualizando de manera coherente todas las referencias a enlaces en el mapa de sitio visual, el menú de navegación y la página de inicio.
- **Redirección 301 en Middleware Edge para /pricing/**: Programado un redireccionamiento permanente HTTP 301 a nivel Edge en `src/hooks.server.ts` de `/pricing` y `/pricing/` hacia `/packages/` para capturar y transferir de forma total el posicionamiento orgánico y PageRank de la web histórica live de WordPress.
- **Transición Semántica de "Pricing" a "Packages" en Menú**: Renombrada la clave global de traducción de `nav.pricing` a `nav.packages` en los diccionarios reactivos de `i18n.svelte.ts` y del catálogo estático `translations.ts` para mostrar reactivamente "Packages" en inglés y "Paquetes" en español en los componentes de navegación móviles y de escritorio.
- **FAQPage SEO Técnico en Catálogo de Paquetes**: Inyectado dinámicamente el esquema estructurado de Schema.org `FAQPage` (JSON-LD) en la página del catálogo de paquetes (`/packages/`), mapeando la pregunta y respuesta localizada de IVA (`vat-pricing`) directamente desde la base de datos centralizada de FAQs, habilitando a Malaga Event Gear para calificar para Rich Snippets en Google.
- **Robustecimiento y Expansión de Tests de Esquemas**: Ampliada la cobertura de pruebas de Playwright en `tests/schema.spec.ts` para validar de forma exhaustiva y preventiva que todas las rutas públicas del sitio (Home `/`, catálogo `/packages/`, todos los planes individuales de `/packages/[slug]/`, servicios `/services/`, preguntas frecuentes `/faq/`, blog `/blog/`, corporativas y políticas legales) inyecten con exactitud matemática los esquemas de Schema.org (`ProfessionalService`, `ItemList`, `Service`, `FAQPage`, `BreadcrumbList`) requeridos por [.agents/STRUCTURED_DATA.md](file:///Users/hlorenzoz/databank/Development/%5BMEG%20-%20Malaga%20Event%20Gear%20%28malagaeventgear.com%29%5D/projects/website/.agents/STRUCTURED_DATA.md).
- **Estrategia de Espera Inteligente Basada en DOM**: Refactorizado el extractor de JSON-LD en Playwright (`getJsonLdSchemas`) para esperar a que los scripts estén presentes en el DOM (`first().waitFor({ state: 'attached' })`) en lugar de esperar la inactividad de red (`waitForLoadState('networkidle')`). Esto eliminó los timeouts crónicos causados por el websocket de HMR de Vite en desarrollo, acelerando la ejecución de la suite de 1.5 minutos a tan solo 22 segundos de manera 100% estable.
- **Visible Glassmorphic Breadcrumbs Component**: Created a reactive Svelte 5 component (`src/lib/components/navigation/Breadcrumbs.svelte`) utilizing the `$derived` rune to dynamically segment the page pathname and translate it on-the-fly based on the active language (`i18n.lang`), with styling conforming strictly to the premium glassmorphic specifications in `DESIGN.md`.
